Artifact Enhancements

1. Software Engineering and Design

Milestone Two Narrative (Word)

To improve modularity and maintainability, I refactored the backend Express API by extracting logic into middleware, adding centralized Winston-based logging, and restructuring routes for clarity. I followed clean architecture principles learned in CS 320 and CS 340 to break up monolithic logic and improve testability.

2. Algorithms and Data Structures

Milestone Three Narrative (Word)

This enhancement focused on performance and clarity. I rewrote the date sorting algorithm, optimized filtering utilities, and removed nested conditional logic. These changes reduced cognitive complexity and made the frontend behavior more predictable and efficient, particularly in how it filters and displays travel entries.

3. Databases

Milestone Four Narrative (Word)

I improved the backend’s security posture and data handling by implementing JWT authentication, redesigning Mongoose schemas to be more scalable, and optimizing database queries. I also added input validation and sanitized all user inputs to protect against injection attacks and reduce the risk of data corruption.

Artifact Overview: Travlr Getaways

  • Frontend: Angular with Bootstrap
  • Backend: Node.js using Express and Handlebars
  • Database: MongoDB via Mongoose
  • Authentication: JSON Web Tokens (JWT)
  • Logging & Error Handling: Winston, custom middleware
